PocketMine-MP là gì, và vì sao nó kéo mình vào Công nghệ thông tin.

Một cánh cửa nhỏ của Minecraft Bedrock, mở ra bằng PHP, plugin và rất nhiều lần nhìn console đỏ lòm rồi tự hỏi mình đã phá cái gì nữa.

← Tất cả bài viết

Nếu phải chọn một thứ dẫn mình vào Công nghệ thông tin theo cách rất đời, mình sẽ chọn PocketMine-MP. Không phải vì nó là thứ “xịn” nhất để chạy server Minecraft Bedrock. Cũng không phải vì nó mô phỏng vanilla hoàn hảo. Nó quan trọng với mình vì nó là nơi mình bắt đầu hiểu server là gì, plugin là gì, config là gì, lỗi console là gì, và vì sao một dấu ngoặc sai có thể làm cả buổi tối mất bình yên.

PocketMine-MP, hay PMMP, là phần mềm server mã nguồn mở dành cho Minecraft Bedrock Edition. Nó được viết chủ yếu bằng PHP, với một số phần bằng C và C++. Trang chính thức mô tả nó như một server có khả năng tùy biến cao cho Bedrock. Repo chính trên GitHub cũng nói thẳng hơn một chút, đây là server software được xây từ đầu, không phải bản chỉnh sửa của server Bedrock gốc.

PocketMine-MP không chỉ là “chạy server Minecraft”. Nó là một hệ sinh thái để bẻ gameplay theo ý mình bằng plugin.

Nó không phải server vanilla

Điều cần nói đầu tiên là PocketMine-MP không phải Minecraft Bedrock Dedicated Server của Mojang. Nó là một implementation riêng, có giao thức, thế giới, entity, item, event, command và API riêng. Vì vậy nó rất hợp để làm server tùy biến, minigame, lobby, kinh tế, rank, quest, tiện ích quản trị hoặc các hệ thống “không giống survival gốc”.

Đổi lại, nếu chỉ muốn mở một server sinh tồn giống Minecraft gốc nhất có thể, PocketMine-MP không phải lựa chọn ngon nhất. Repo chính của PMMP cũng cảnh báo rõ chuyện này. Các thứ như world generation kiểu vanilla, redstone, AI mob và nhiều cơ chế gameplay gốc có thể thiếu, khác hoặc cần plugin bù vào.

Nói ngắn hơn, nếu cần “giống Minecraft nhất”, hãy cân nhắc server chính thức. Nếu cần “server Bedrock nhưng mình muốn sửa luật chơi”, PocketMine-MP bắt đầu đáng nhìn.

Vì sao PHP lại xuất hiện ở đây?

Nhìn từ ngoài vào, PHP nghe hơi lạ. Người mới thường nghĩ server game phải là C++, Java, Go hoặc thứ gì đó “ngầu” hơn. Nhưng với cộng đồng PocketMine-MP, PHP lại là một điểm rất dễ bén duyên. Cài server, mở file plugin, sửa vài dòng, reload, nhìn lỗi, sửa tiếp. Vòng lặp đó nhanh và khá thân thiện với người mới.

Điểm hay của PMMP nằm ở API. Plugin có thể bắt sự kiện người chơi vào server, chat, đặt block, phá block, dùng item, chạy command, đổi permission, thao tác world, gửi form, quản lý task và lưu dữ liệu. Một server Bedrock bình thường từ đó có thể thành server minigame, server sinh tồn có luật riêng, hoặc một đống thử nghiệm mà chủ server nghĩ ra lúc ba giờ sáng.

Nhưng PHP dễ bắt đầu không có nghĩa là dễ làm tốt. Plugin chạy trên server thật phải quan tâm đến tick, đọc ghi dữ liệu, task bất đồng bộ, bộ nhớ, dữ liệu người chơi và chuyện tương thích API sau mỗi major version. PMMP dạy người ta bằng cách khá thẳng tay. Sai thì console nói. Nói hơi dài. Đôi lúc nói như mắng.

Hệ sinh thái quanh PMMP

PocketMine-MP không sống một mình. Nó có docs, API docs, GitHub, Discord, Poggit, DevTools, ExamplePlugin, virion và rất nhiều plugin cộng đồng.

Poggit là nơi nhiều người biết đến nhất khi tìm plugin. Nó vừa là kho phát hành plugin đã qua review, vừa là hệ thống build từ GitHub để tạo file phar. Với người dùng server, Poggit là nơi tải plugin. Với dev, Poggit là chỗ build, kiểm tra và phát hành plugin theo cách đỡ thủ công hơn.

DevTools lại là món gần như ai từng viết plugin cũng chạm qua. Nó có thể tạo khung plugin, load plugin dạng folder khi dev, build plugin thành phar và làm vài việc nhỏ khác. ExamplePlugin thì đúng như tên, một plugin mẫu để nhìn cách API hoạt động. API docs được sinh lại theo release, dùng để tra class, method và signature khi code bắt đầu lớn hơn trí nhớ của mình.

Virion là một khái niệm khá riêng của hệ Poggit. Có thể xem nó như thư viện nhỏ dành cho plugin PocketMine-MP, được đóng gói cùng plugin khi build. Nhờ vậy nhiều plugin có thể dùng chung một phần code mà không bắt người cài server phải tự xử lý dependency như một project PHP truyền thống.

Version luôn chạy theo Bedrock

Một nỗi khổ rất quen của thế giới Minecraft Bedrock là game update liên tục. Mỗi lần Bedrock đổi protocol, server phải chạy theo. PocketMine-MP cũng vậy. Cộng đồng thường chờ bản PMMP mới để server tương thích với client mới, còn dev plugin thì chờ xem API có gì đổi không.

Mốc mình kiểm tra khi viết bài này là ngày 30.06.2026. Release mới nhất trên GitHub đang là PocketMine-MP 5.44.2, phát hành ngày 24.06.2026, dành cho Minecraft Bedrock 1.26.30, display version v26.30. Release này cũng nhắc đến PHP binary khuyến nghị cho nhánh PM5 là PHP 8.2.

Chi tiết version nhìn hơi khô, nhưng nó là đời sống thật của server Bedrock. Một buổi sáng người chơi update game, buổi chiều server chưa vào được, tối admin đọc changelog, còn dev plugin cầu mong API không đổi quá đau.

Điểm mạnh của PocketMine-MP

Điểm mạnh lớn nhất là tùy biến. PocketMine-MP được sinh ra cho người muốn can thiệp sâu vào server. Plugin API làm cho việc đổi gameplay trở thành một hoạt động bình thường, không phải một nghi lễ gọi thần.

  • Muốn làm command riêng, được.
  • Muốn chặn một hành vi của người chơi, được.
  • Muốn làm hệ rank, coin, lobby, minigame, form, item đặc biệt, được.
  • Muốn lưu dữ liệu người chơi rồi đồng bộ với web hoặc bot, cũng được, miễn là code không tự bắn vào chân.

Điểm mạnh thứ hai là cộng đồng. Không phải cộng đồng lúc nào cũng hoàn hảo, nhưng nó đủ lớn để người mới tìm được plugin, ví dụ, câu trả lời cũ và đôi khi là một người lạ chịu chỉ ra lỗi rất bé trong file plugin.yml.

Điểm mạnh thứ ba là cảm giác sở hữu. Khi chạy PocketMine-MP, server không còn là một hộp đen. Mình có thể mở code, đọc API, viết plugin, sửa plugin, fork plugin, giữ lại plugin sợ mất, hoặc làm cả một tổ chức GitHub chỉ để lưu trữ mấy thứ có thể biến mất. Nghe hơi ám ảnh, nhưng đúng là mình đã làm vậy.

Điểm yếu cũng nên nói thẳng

PocketMine-MP không phải cây đũa thần. Nó không biến một server Bedrock thành mọi thứ cùng lúc. Nếu plugin viết kém, server lag. Nếu admin cài bừa, server loạn. Nếu kỳ vọng survival vanilla hoàn hảo, dễ thất vọng.

Một điểm yếu khác là sự phụ thuộc vào plugin cộng đồng. Nhiều plugin rất tốt, nhiều plugin bị bỏ dở, nhiều plugin chỉ chạy ở API cũ, nhiều plugin từng là cứu tinh một thời rồi sau đó nằm im như hóa thạch. Đây cũng là lý do mình có các kho lưu trữ plugin và fork. Internet không hứa giữ giùm mình thứ gì cả.

Với người mới, PHP và API PMMP dễ vào, nhưng debug server thật vẫn là một bài học. Crash không phải lúc nào cũng rõ. Lỗi tương thích version đôi khi trông giống lỗi của mình, nhưng thật ra đến từ plugin cũ. Và có những bug chỉ xuất hiện khi vài chục người chơi cùng làm chuyện mà dev không ngờ tới.

Nó đã ảnh hưởng tới mình thế nào?

Mình đến với Minecraft từ khoảng năm 2015, rồi dần dính vào Minecraft Pocket Edition, Bedrock server, plugin và PocketMine-MP. Ban đầu chỉ là tải game, chơi game, mở server cho vui. Sau đó là config, mở port, backup dữ liệu, sửa lỗi console, tìm plugin, Việt hóa, mò Poggit, đọc code người khác và viết những plugin đầu tiên.

Plugin PocketMine-MP chính là thứ bén duyên để mình học Công nghệ thông tin. Nó không bắt đầu bằng giáo trình. Nó bắt đầu bằng nhu cầu rất cụ thể. Mình muốn server làm được một việc gì đó, plugin chưa có hoặc có mà không đúng ý, thế là phải sửa. Sửa không được thì học. Học xong thì lại muốn sửa tiếp. Một vòng lặp khá nguy hiểm cho thời gian ngủ.

Sau này mình có NhanAZ-Plugins để lưu plugin, NhanAZ-Libraries để giữ virion và thư viện nhỏ, NhanAZ-Web cho mấy web tool, pm-pl để lưu plugin PocketMine-MP vì sợ tài nguyên cộng đồng biến mất, và PMMPVN như một mảnh ký ức của cộng đồng PocketMine-MP Việt Nam.

Năm 2019, mình lập group PocketMine-MP Việt Nam. Group không lớn, hơn 350 thành viên, nhưng vẫn còn đó. Với mình, nó giống một góc nhỏ cho những người từng loay hoay với server Bedrock, plugin, API, lỗi console và cảm giác vui rất lạ khi một thứ tự mình sửa cuối cùng cũng chạy.

Nếu hôm nay mới bắt đầu

Nếu ai đó mới chạm PocketMine-MP, mình nghĩ nên bắt đầu bằng một server sạch, ít plugin, đọc docs cài đặt trước, rồi cài từng plugin một. Đừng tải một pack lạ, ném vào server, bật lên thấy chạy rồi nghĩ là ổn. Nó có thể ổn thật. Hoặc nó đang chờ tới lúc đông người để trả thù.

Nếu muốn viết plugin, hãy xem ExamplePlugin, đọc API docs, thử event đơn giản, tạo command nhỏ, lưu một file config, rồi mới nghĩ đến hệ thống lớn hơn. Plugin đầu tiên không cần vĩ đại. Nó chỉ cần giúp mình hiểu vòng đời plugin, cách PMMP gọi event, và vì sao server main thread không thích bị bắt làm việc nặng.

Nếu muốn chạy server nghiêm túc, hãy học backup, log, quyền file, giới hạn tài nguyên, version PHP, version PMMP, version plugin và cách đọc changelog. Server Minecraft vui, nhưng dữ liệu người chơi thì không phải đồ chơi.

Vì sao mình vẫn thấy nó đáng giữ lại?

PocketMine-MP là một dự án hơi đặc biệt. Nó đứng giữa game, cộng đồng, open source, plugin, reverse engineering nhẹ nhàng ở tầng giao thức, và nhu cầu rất đời của những người muốn có một server riêng. Nó không hoàn hảo, nhưng nó đủ mở để người ta học bằng cách làm.

Với mình, PMMP là một phần của ký ức kỹ thuật. Nó dạy mình rằng code không phải thứ nằm trong sách. Code có thể là một plugin nhỏ làm người chơi nhận thưởng mỗi ngày. Là một lỗi crash khiến server không mở được. Là một repo bị xóa nên mình tiếc. Là một tool viết ra vì không muốn phụ thuộc vào một trang có thể biến mất.

Và chắc cũng vì vậy mà tới giờ, mỗi lần nhìn thấy chữ PocketMine-MP, mình vẫn có cảm giác rất rõ. Đây không chỉ là server software. Đây là một con đường nhỏ đã dẫn mình tới rất nhiều thứ khác.
