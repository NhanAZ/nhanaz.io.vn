GitHub

The shelves where I keep plugins, tools, forks, static web utilities, archives, and projects that were once more ambitious than practical.

A small map

More than one profile.

My GitHub is split across many organizations because each shelf has a different job. Some hold PocketMine-MP plugins. Some preserve things I am afraid might disappear. Some cannot be opened freely because copyright and legal risk are real constraints. A few shelves are still active, a few are dusty, and a few are simply proof that I once aimed much bigger than I could carry.

I gathered them here so the organizations do not look like random fragments. Each one keeps a different kind of memory, from things I built and things I archive for the community to ideas that stopped halfway.

The rest of NhanAZ

Side drawers

Not every shelf needs its own district. Some are still interesting, like data, tools, and dropped projects. The rest is grouped by purpose so this page does not turn into an org directory.

NhanAZ-DataA newer shelf for datasets, collection work, statistics, and numbers that look dry until they start telling a story.Data and small research

I like collecting data because every table carries a different kind of story. Some projects are for research, some are for prediction experiments, and some simply turn raw data into something easier to read. This drawer is for curiosity that can be measured.

GitHub org ↗
NhanAZ-ToolsTools, reverse engineering notes, encryption, decryption, unofficial APIs, and work that is fun to build but risky to release carelessly.Sharp technical tools

Many things here sit close to the line between technical curiosity and real risk. I like taking systems apart to understand how they work, but making something possible does not mean it should be published immediately. Some tools may open later. Some should probably stay in the drawer.
